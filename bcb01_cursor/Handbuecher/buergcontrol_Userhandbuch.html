Finanzielle Parameter

Grundlegende Werte für die Bürgschaftsberechnung

Bürgschaftserhöhung (optional)

Parameter im Detail:

Bewilligungszeitraum
Von-Datum: Beginn des genehmigten Zeitraums für Ihre Bürgschaft
Bis-Datum: Ende des Bewilligungszeitraums
Wichtig: Nur Vorgänge mit Gestellungsdatum in diesem Zeitraum werden verarbeitet!
Bürgschafts-Startsumme
Was ist das? Die von Ihrer Bank/Versicherung bereitgestellte Bürgschaftssumme
Wo finde ich das? In Ihrem Bürgschaftsschein
Format: Euro mit 2 Dezimalstellen
Beispiel: 5.000.000,00 € (5 Millionen Euro)
Ersatz-Zollsatz bei 0%
Zweck: Wird automatisch verwendet, wenn Waren mit 0% Zollsatz gefunden werden
Empfehlung: 12% (durchschnittlicher EU-Zollsatz)
Warum? Für realistische Bürgschaftswerte bei zollfreien Waren (z.B. Rohkaffee)
Berechnung: Warenwert × 12% = Bürgschaftsbelastung
Pauschalbetrag
Zweck: Bürgschaftsbetrag für Positionen ohne ermittelbaren Zollwert
Standard: 10.000 € (vom Zoll akzeptiert)
Wann verwendet? Bei "KEIN MATCH" oder leeren Anmeldearten
Effekt: Verhindert 0€-Einträge in der Berechnung
Bürgschaftserhöhung (optional)
Wann nutzen? Für unterjährige Erhöhungen der Bürgschaftssumme
Beispiel: Bei Geschäftswachstum oder saisonalen Spitzen
Datum: Ab wann gilt die erhöhte Bürgschaft?
Betrag: Um welchen Betrag wurde die Bürgschaft erhöht?
Effekt: Automatische Berücksichtigung in der Tagessaldo-Berechnung

Mandantenverwaltung

buergcontrolBASE ist mandantenfähig - jedes Unternehmen hat:

  • Eigenen Aktivierungscode - eindeutige Kennung
  • Eigene Benutzer - separate Login-Daten
  • Eigene Konfiguration - gespeichert als settings_[mandant].json
  • Eigene Historie - gespeichert als historie_[mandant].json
  • Optional: Eigenes Logo - wird in Sidebar und Dokumenten angezeigt

Automatische Optimierungen:
  • EZA-Spalten-Reduzierung: Automatisch auf 13 Standard-Spalten
  • Duplikate-Entfernung: Gleiche MRN + Position werden entfernt
  • BE-Anteil-Aufteilung: "ATB123 - POS 1" → separate Zeilen

4.2 Import ZL / VAV (für WIDS)

Was ist WIDS? Zolllager-/VAV-Export mit voraussichtlichen Abgaben.
Export aus: ATLAS → Zolllager → Vorgänge ODER VAV → Bewegungen
Besonderheit: System erkennt Tippfehler in Spaltennamen!
Schreibweisen-Toleranz:
Erwartet Auch erkannt
Registriernummer/MRN Registrienummer/MRN

DV1UmgerechnerterRechnungsbetrag DV1 Umgerechneter Rechnungsbetrag
WIDS-Aggregation bei mehreren Positionen:
Standard-Einstellung: "Position mit höchstem Zollwert"
• System wählt automatisch die wertvollste Position
• Anzeige: "Pos 3 (max von 5)" im Ergebnis
Alternativen: Nur Position 1, Summe aller Positionen

Matching-Algorithmus

IMDC - 3-Kriterien-Matching

1. Suche MRN in "Weitere Registriernummer"
2. Wenn BE-Anteil vorhanden:
   - Prüfe MRN + ATB + Position
   - Beispiel: 24DEABC123 + 24DEATB456 + "1"
3. Kein präziser Match → Fallback auf MRN
4. Kein Match → Pauschalbetrag

WIDS - Aggregations-Modi

Wenn mehrere Positionen pro MRN:

1. "Position mit höchstem Zollwert" (Standard)
   → Wählt wertvollste Position
   → Anzeige: "3 (max von 5)"

2. "Nur Position 1"
   → Nimmt erste Position
   → Anzeige: "1 (1 von 5)"

3. "Summe aller Positionen"
   → Addiert alle Zollwerte
   → Anzeige: "SUMME (5 Pos.)"

Berechnungslogik

Zollwert-Berechnung bei WIDS

WENN Zollsatz = 0%:
    Zollwert = DV1-Betrag
SONST:
    Zollwert = Zollabgabe ÷ (Zollsatz ÷ 100)

Bürgschaftssaldo-Berechnung

Für jeden Tag:
    Belastung = Summe aller Eingänge
    Entlastung = Summe aller Ausgänge
    Netto = Entlastung - Belastung
    Neuer Stand = Alter Stand - Belastung + Entlastung
    
    Tiefststand = MIN(alle Stände des Tages)
    Höchststand = MAX(alle Stände des Tages)
    
Auslastung% = (Startbürgschaft - Globaler Tiefststand) ÷ Startbürgschaft × 100

Geschäftsregeln im Detail

Regel: Mindestabgaben

WENN Gesamtabgaben > 0 UND < 1:
    Gesamtabgaben = 1
GRUND: Verwaltungsaufwand rechtfertigt Mindestbetrag

Regel: Null-Euro-Behandlung

WENN Gesamtabgaben = 0:
    WENN Zollwert > 0:
        Gesamtabgaben = 1
    SONST:
        Gesamtabgaben = Pauschalbetrag
GRUND: Jede Position muss Bürgschaft belasten

Regel: ATB-Filter

WENN "Weitere Registriernummer" beginnt mit "ATB":
    Zeile überspringen
    Zähler erhöhen
GRUND: Interne Umbuchungen ohne echte Warenbewegung

Mögliche Ursachen:
  1. Falsches Datumsformat: Muss DD.MM.YYYY sein (nicht MM/DD/YYYY)
  2. Zeitraum zu klein: Erweitern Sie Von/Bis-Datum
  3. Spalte "Datum Überlassung - CUSTST" leer
  4. Alle Daten außerhalb des Zeitraums

Bedeutung: Sie haben IMDC-Zeilen, aber keine EZA-Datei hochgeladen
Lösung:
  1. Prüfen Sie den Zeitraum in der Info-Box
  2. Exportieren Sie aus ATLAS für diesen Zeitraum
  3. Laden Sie die Datei unter "4.1 Import EZA" hoch

Datenqualitätsprobleme

Viele "KEIN MATCH" Einträge

Ursache Erkennung Lösung
MRN-Format unterschiedlich Leitdatei: 24DEABC123
Import: 24DEABC123.01		 System bereinigt automatisch ".XX" Endungen
Zeitversatz Gestellung: 31.03.2024
Verzollung: 02.04.2024		 Import-Dateien für beide Monate laden
BE-Anteil fehlt Spalte 13 nicht in EZA Original-Export mit allen Spalten verwenden
Falsche MRN-Spalte Weitere vs. normale Registriernummer System prüft beide automatisch

Bürgschaft wird negativ

Das ist normal wenn:
• Mehr Ausgänge als Eingänge im Zeitraum
• Altbestände aus Vorperiode abgebaut werden
• Fehler in Vorperiode korrigiert werden

Prüfen Sie:
• Ist die Startbürgschaft korrekt?
• Sind alle Eingänge erfasst?
• Ist der Zeitraum vollständig?

Performance-Probleme

Verarbeitung dauert sehr lange

  • Große Dateien: Zeitraum verkleinern (monatsweise)
  • Browser-Problem: Cache leeren oder anderen Browser
  • Viele BE-Anteile: Normal, da Zeilen multipliziert werden
  • Empfehlung: Max. 10.000 Zeilen pro Verarbeitung

Download funktioniert nicht

  • Popup-Blocker: Deaktivieren für diese Seite
  • Speicherplatz: Download-Ordner prüfen
  • Große Datei: Geduld, kann 10-30 Sekunden dauern
  • Alternative: Im Bürgschaftsverlauf neu generieren

Spezialfälle

Stornierungen und Korrekturen

Negative Zollwerte in Import-Dateien:
• System verarbeitet diese normal
• Negative Werte reduzieren Bürgschaft
• Sichtbar in Tagessaldo als negative Belastung

Teilerledigungen

Eine ATB-Position, mehrere Erledigungen:
• Jede Erledigung = eigene Zeile in Leitdatei
• Teilmengen werden automatisch verarbeitet
• Bürgschaft wird stufenweise entlastet

Sammelanmeldungen

Eine Verzollung für mehrere ATB:
• System sucht alle passenden ATB
• Bei WIDS: Verteilung nach Konfiguration
• Sonst: Pauschale für nicht zuordenbare

🎯 Was ist buergcontrol?

buergcontrol ist eine Anwendung zur sicheren und nachvollziehbaren Berechnung der Bürgschaftsauslastung im Verwahrungslagerverfahren – speziell für Zollverantwortliche, Logistikleiter und Controller.

  • Ermöglicht tagesgenaue Überwachung der Sicherheitswerte
  • Berechnet alle Zollabgaben automatisch auf Basis gelieferter Daten
  • Erzeugt prüferkonforme Export-Dokumentation (Excel, Word)

📘 Praxisbeispiel: So läuft es ab

  1. Mandant wählen: Demo GmbH
  2. Zeitraum wählen: 01.05.2024 bis 31.05.2024
  3. SumA-Leitdatei hochladen: demo_suma.csv
  4. EZA-Importdatei ergänzen (optional)
  5. „Verarbeiten“ klicken
  6. Statistik prüfen, dann auf „Export Excel + Word“ klicken

🧯 Fehlerhilfe & häufige Probleme

  • Datei nicht erkannt: → Dateiformat prüfen (.csv, .xlsx)
  • „Keine Daten im Zeitraum“: → Zeitraum prüfen (Daten vorhanden?)
  • „Pflichtspalte fehlt“: → Spaltenüberschriften prüfen (Groß-/Kleinschreibung!)
  • BE-Fall nicht erkannt: → Importdatei für EZA/BEs hinzufügen
